# CLKN Whole-App Audit — Findings to Review

**Date:** 2026-07-06 · **Method:** 14-dimension multi-agent audit (find → adversarial verify → synthesis), 46 agents. Findings below survived verification (verify pass: 46 CONFIRMED / 2 PLAUSIBLE / 2 REFUTED out of 50).

> ⚠️ **Note:** the workflow's auto-synthesis returned a false "all clean" — a resume glitch fed it an empty list. This doc is rebuilt from the raw agent findings. The HIGH items were re-verified by hand against the code (notes inline).

**Headline:** No theft-class bug (nothing drains a wallet or bypasses auth to money). Real issues are: fail-open durability bugs (payment-replay + data-loss), live-position-safety bugs (anchor auto-close, paused-revert), and lock-pipeline edge cases. None on fire; several worth fixing soon.

## Recommended fix order (my suggestion)
1. **Durability cluster** (#1–#3 below, HIGH) — atomic temp+rename writes + fail-closed on corrupt file. Protects payments, diplomas, money ledgers. No behavior change, low risk.
2. **Anchor exclusion** (#4–#5, HIGH) — stop the vault adopting/closing permanent anchors. Protects live positions. One line each.
3. **Paused-vault reverts** (medium) — Blitz/heavy revert must respect the pause (STOP-means-stay-stopped).
4. **Lock-celebration hardening** (medium cluster) — the pipeline is live, so worth doing.
5. Everything else as time allows.

### 20. sigstore persist() uses a non-atomic writeFileSync and load() keeps persistent=true after a corrupt-file read, so a crash-corrupted signatures file silently wipes the consumed-set on boot and re-enables replay. — ✅ hand-verified
- **Where:** `lib/sigstore.js:23`  ·  _Payment & replay path_
